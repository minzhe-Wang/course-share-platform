# 项目使用指南

本文档面向第一次接触本项目的同学，说明需要安装什么软件、如何从 GitHub 拉取项目、如何使用 Docker 部署、如何进入开发模式，以及遇到问题应该看哪个文档。

## 1. 项目技术栈

本项目是一个课程资料共享平台：

| 层次 | 技术 |
| --- | --- |
| 前端 | Vue 3、Vite |
| 后端 | Spring Boot、MyBatis |
| 数据库 | MySQL 8 |
| 缓存 | Redis |
| 文件存储 | MinIO |
| 网关 | Nginx |
| 部署 | Docker Compose |

项目支持两种运行方式：

| 模式 | 适合场景 | 特点 |
| --- | --- | --- |
| 部署模式 | 老师验收、同学快速运行、课程展示 | 前端、后端、数据库、Redis、MinIO、Nginx 全部由 Docker 启动 |
| 开发模式 | 修改代码、调试接口、日常开发 | Docker 只启动依赖服务，前端和后端在本地运行 |

## 2. 需要安装的软件

### 2.1 Git

作用：从 GitHub 拉取项目、提交代码。

官方下载地址：

```text
https://git-scm.com/install/windows
```

安装后检查：

```powershell
git --version
```

### 2.2 Docker Desktop

作用：启动 MySQL、Redis、MinIO、后端、前端和 Nginx。

官方下载地址：

```text
https://www.docker.com/products/docker-desktop/
```

安装建议：

- Windows 推荐使用 WSL 2 后端。
- 安装完成后先启动 Docker Desktop。
- 等 Docker Desktop 显示运行正常后再执行项目命令。

检查命令：

```powershell
docker --version
docker compose version
```

### 2.3 Node.js

作用：前端本地开发时运行 Vue/Vite。

官方下载地址：

```text
https://nodejs.org/en/download
```

建议安装 LTS 版本。

检查命令：

```powershell
node -v
npm -v
```

作用：后端本地开发时运行 Spring Boot。

推荐 Eclipse Temurin JDK 17：

```text
https://adoptium.net/temurin/releases/?version=17
```

检查命令：


正常应看到 Java 17。

安装说明：

检查命令：

## 4. 部署模式

部署模式适合老师验收和同学快速运行。

### 4.1 复制环境配置

```powershell
Copy-Item .env.example .env
```

`.env` 是本地环境配置，不要提交到 GitHub。

### 4.2 启动全部服务

```powershell
docker compose up -d --build
```

第一次启动会下载镜像并构建前后端，时间会比较久。

### 4.3 查看服务状态

```powershell
docker compose ps
```

正常应看到：

```text
course_mysql      healthy
course_redis      healthy
course_minio      healthy
course_backend    healthy
course_nginx      healthy
```

### 4.4 访问地址

| 功能 | 地址 |
| --- | --- |
| 前端页面 | `http://localhost` |
| 后端 API 统一入口 | `http://localhost/api` |
| 后端直连调试 | `http://localhost:18080/api` |
| 健康检查 | `http://localhost/api/health` |
| MinIO 控制台 | `http://localhost:9001` |

MinIO 默认账号：

```text
minioadmin / minioadmin
```

默认业务账号：

```text
student / 123456
reviewer / 123456
admin / 123456
```

### 4.5 运行冒烟测试

```powershell
powershell -ExecutionPolicy Bypass -File .\backend\scripts\api-smoke-test.ps1
```

看到下面输出说明部署成功：

```text
Smoke test passed.
```

## 5. 开发模式

开发模式适合改代码。推荐做法是：

```text
Docker 只运行 MySQL / Redis / MinIO
后端本地 Maven 启动
前端本地 npm 启动
```

### 5.1 启动依赖服务

在项目根目录执行：

```powershell
docker compose up -d mysql redis minio minio-init
```

如果之前已经启动完整部署模式，可以先停掉 Docker 后端和 Nginx：

```powershell
docker compose stop backend nginx
```

### 5.2 启动后端

打开一个新的 PowerShell：

```powershell
cd course-share-platform\backend
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

后端开发地址：

```text
http://localhost:8080
```

验证：

```text
http://localhost:8080/api/health
http://localhost:8080/api/test
http://localhost:8080/api/db/time
http://localhost:8080/api/redis/test
```

### 5.3 启动前端

再打开一个新的 PowerShell：

```powershell
cd course-share-platform\frontend
npm install
npm run dev
```

前端开发地址：

```text
http://localhost:5173
```

开发模式链路：

```text
浏览器
-> http://localhost:5173
-> Vite 前端开发服务器
-> 本地 Spring Boot 后端 http://localhost:8080
-> Docker MySQL / Redis / MinIO
```

## 6. 常见配置修改

配置文件是项目根目录下的 `.env`。

如果端口冲突，优先改 `.env`，不要直接改 `docker-compose.yml`。

### 6.1 Nginx 80 端口冲突

```text
NGINX_PORT=8088
```

访问地址改为：

```text
http://localhost:8088
```

### 6.2 MySQL 3306 端口冲突

```text
MYSQL_PORT=13306
```

### 6.3 Redis 6379 端口冲突

```text
REDIS_PORT=16379
```

### 6.4 MinIO 端口冲突

```text
MINIO_API_PORT=19000
MINIO_CONSOLE_PORT=19001
```

### 6.5 后端直连端口冲突

```text
BACKEND_PORT=18081
```

后端直连地址改为：

```text
http://localhost:18081/api
```

修改 `.env` 后重新启动：

```powershell
docker compose up -d
```

## 7. 常见问题

### 7.1 docker compose ps 不是 healthy

先查看日志：

```powershell
docker compose logs -f backend
docker compose logs -f mysql
docker compose logs -f nginx
```

然后根据日志排查端口、数据库连接、MinIO bucket、依赖启动顺序等问题。

### 7.2 页面能打开，但接口失败

先访问健康检查：

```text
http://localhost/api/health
```

如果 `mysql`、`redis` 或 `minio` 是 `DOWN`，优先查看对应服务日志。

### 7.3 数据库初始化数据不对

MySQL 初始化脚本只在 Docker volume 首次创建时执行。如果改了 `docker/mysql/init.sql`，需要重置 volume：

```powershell
docker compose down -v
docker compose up -d --build
```

注意：`down -v` 会删除本地数据库和 MinIO 数据，只适合本地开发或课程演示环境。

### 7.4 PowerShell 里中文显示乱码

如果接口返回在浏览器中是中文，但 PowerShell 显示乱码，一般是终端编码问题，不一定是后端或数据库坏了。

可以优先用浏览器打开接口验证：

```text
http://localhost/api/categories
http://localhost/api/tags
```

项目维护者修改代码后：

```powershell
git status

docker compose ps
powershell -ExecutionPolicy Bypass -File .\backend\scripts\api-smoke-test.ps1
```

```powershell
git add -A
git status
git commit -m "complete docker deployment and development guide"
git push origin main
```
