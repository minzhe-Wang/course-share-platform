# 演示流程

本文档用于课堂验收、录屏和同学复现演示。建议先跑 Docker 部署，再用浏览器完成学生端核心闭环。

## 1. 演示前准备

进入项目根目录：

```powershell
cd D:\dev\Projects\course-share-platform
```

启动全部服务：

```powershell
docker compose up -d --build
```

查看容器状态：

```powershell
docker compose ps
```

确认以下服务都是 `healthy`：

```text
course_mysql
course_redis
course_minio
course_backend
course_nginx
```

运行冒烟测试：

```powershell
powershell -ExecutionPolicy Bypass -File .\backend\scripts\api-smoke-test.ps1
```

看到 `Smoke test passed.` 说明后端、数据库、Redis、MinIO 和 Nginx 联调可用。

## 2. 打开前端

浏览器访问：

```text
http://localhost
```

页面是课程资料共享平台的核心演示工作台，包含：

```text
资料库
智能推荐
发布资料
问答区
发布问题
账号登录/注册
```

左侧导航用于切换模块，顶部展示服务状态、当前页面和关键统计。

## 3. 展示健康检查

浏览器访问：

```text
http://localhost/api/health
```

重点说明：

```text
application = UP
mysql = UP
redis = UP
minio = UP
```

这个接口证明后端和基础设施依赖已经联通。

## 4. 登录学生账号

回到前端页面，进入“账号”：

```text
student / 123456
```

说明点：

```text
当前项目第一版使用 mock-token-{userId} 作为登录凭证。
前端登录后会保存 token，并在后续请求中携带 Authorization 请求头。
```

## 5. 发布正常资料

进入“发布资料”，填写示例：

```text
标题：软件工程课程设计示例资料
简介：用于课堂演示的资料记录，内容正常，会通过词库审核。
分类：软件工程
标签：课程设计、课件等
文件类型：PDF / DOC / DOCX / ZIP
```

提交后说明：

```text
后端会校验用户身份、分类、标签和文件信息。
随后调用词库审核服务。
未命中敏感词时审核结果为 PASS，资料状态改为 APPROVED。
```

## 6. 展示词库审核拒绝

再次发布一条包含敏感词的资料或问题，例如：

```text
标题：广告资料测试
简介：这里包含诈骗或广告内容
```

预期结果：

```text
词库审核命中敏感词，返回 REJECT 或 RISK。
业务状态会映射为 REJECTED，不会出现在公开资料列表或问答列表。
```

说明点：

```text
当前验收版本不调用真实大模型，避免 token、网络和费用问题。
系统保留 AiAuditService 接口，后续可以替换为真实内容安全服务。
```

## 7. 查看资料库和详情

进入“资料库”，点击查询或刷新。

说明点：

```text
资料库只展示 audit_status = APPROVED 且 status = 1 的资料。
未审核、审核拒绝、下架内容不会出现在公开列表。
```

点击“查看详情”，展示：

```text
分类
标签
上传者
文件类型
文件大小
浏览数
下载数
点赞与收藏操作
```

访问详情后，浏览数会自动递增。

## 8. 展示智能推荐

进入“智能推荐”。

说明点：

```text
智能推荐基于热度、收藏、下载、浏览和时间因素生成，不消耗外部 AI token。
资料推荐分 = 下载数 * 3 + 收藏数 * 2 + 点赞数 * 2 + 浏览数 + 时间加分。
问题推荐分 = 回答数 * 3 + 点赞数 * 2 + 浏览数 + 时间加分。
```

推荐卡片会展示：

```text
标题
分类
推荐分
推荐理由
```

推荐理由包括：

```text
近期热度较高
收藏和下载较多
讨论活跃
适合优先查看
```

## 9. 展示问答区

进入“发布问题”，填写示例：

```text
标题：软件工程课程设计怎么选题？
内容：想做一个课程资料共享平台，有哪些模块适合作为课程设计？
分类：软件工程
```

提交后说明：

```text
问题同样会经过词库审核。
审核通过后进入问答区公开展示。
```

进入“问答区”，点击问题详情，可以提交回答并查看回答列表。

## 10. 展示部署结构

可以打开以下文件说明工程化部署：

```text
docker-compose.yml
docs/Docker部署教程.md
docs/项目使用指南.md
```

讲解重点：

```text
Nginx 提供前端静态资源并代理 /api。
Spring Boot 后端提供业务接口。
MySQL 保存业务数据。
Redis 用于缓存与连通性验证。
MinIO 保存上传资料文件。
```

## 11. 推荐讲解顺序

```text
1. 介绍项目背景：课程资料共享和互助问答平台。
2. 展示 Docker 一键启动。
3. 打开前端工作台。
4. 展示健康检查，证明依赖服务正常。
5. 登录 student。
6. 发布正常资料，展示词库审核通过。
7. 发布敏感内容，展示词库审核拒绝。
8. 展示资料列表和详情。
9. 展示智能推荐。
10. 展示问答发布和回答。
11. 展示部署教程和项目文档。
```

## 12. 常见问题排查

页面打不开：

```powershell
docker compose ps
docker compose logs -f nginx
```

接口失败：

```powershell
docker compose logs -f backend
```

数据库异常：

```powershell
docker compose logs -f mysql
```

数据不对或中文乱码：

```powershell
docker compose down -v
docker compose up -d --build
```

注意：`down -v` 会删除本地 Docker 数据卷，只适合演示环境重置。
