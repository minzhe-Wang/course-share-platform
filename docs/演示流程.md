# 演示流程

本文档用于课堂展示、项目验收和录屏演示。目标是按一个清晰流程展示：项目可以启动、前端可以访问、后端接口正常、数据库/缓存/文件服务联通，并能完成核心业务闭环。

## 1. 演示前准备

进入项目根目录：

```powershell
cd D:\dev\Projects\course-share-platform
```

启动 Docker：

```powershell
docker compose up -d --build
```

查看容器：

```powershell
docker compose ps
```

确认以下服务都是 `healthy`：

```text
course_mysql
course_redis
course_minio
course_backend
course_nginx
```

运行冒烟测试：

```powershell
powershell -ExecutionPolicy Bypass -File .\backend\scripts\api-smoke-test.ps1
```

看到：

```text
Smoke test passed.
```

说明演示环境可用。

## 2. 展示项目入口

浏览器打开：

```text
http://localhost
```

说明：

```text
这是课程资料共享平台的前端入口。
当前页面由 Nginx 提供静态资源，并通过 /api 反向代理访问后端。
```

可以指出左侧导航：

```text
资料库
发布资料
问答区
发布问题
账号
```

## 3. 展示健康检查

浏览器打开：

```text
http://localhost/api/health
```

说明：

```text
这个接口用于检查后端、MySQL、Redis、MinIO 是否正常。
```

重点展示：

```text
application = UP
mysql = UP
redis = UP
minio = UP
```

## 4. 展示基础接口

浏览器依次打开：

```text
http://localhost/api/test
http://localhost/api/db/time
http://localhost/api/redis/test
```

说明：

```text
/api/test 证明后端服务可访问。
/api/db/time 证明后端可以访问 MySQL。
/api/redis/test 证明后端可以访问 Redis。
```

## 5. 展示基础数据

浏览器打开：

```text
http://localhost/api/categories
http://localhost/api/tags
```

说明：

```text
分类和标签数据来自 MySQL 初始化脚本 docker/mysql/init.sql。
这些数据用于资料筛选和发布资料时选择分类、标签。
```

## 6. 展示登录

回到前端页面：

```text
http://localhost
```

进入账号区域，使用测试账号登录：

```text
student / 123456
```

说明：

```text
第一阶段使用 mock-token 方案。
登录成功后，后端返回 mock-token-{userId}。
前端后续请求会携带 Authorization 请求头。
```

## 7. 展示资料发布

进入“发布资料”页面。

填写示例：

```text
标题：软件工程课程设计示例资料
简介：用于课堂演示的资料记录，内容正常，会通过 Mock AI 审核。
分类：软件工程
标签：大三、课件、课程设计
文件类型：PDF
```

提交后说明：

```text
资料发布时，后端会校验用户身份、分类、标签和文件信息。
随后调用 Mock AI 审核服务。
如果审核通过，资料状态变为 APPROVED，可以在资料列表中展示。
```

## 8. 展示资料列表

回到“资料库”页面。

点击刷新或查询。

说明：

```text
资料列表只展示审核通过并且状态正常的资料。
未审核、审核拒绝、已下架的资料不会显示。
```

展示刚发布的资料：

```text
标题
简介
分类
文件类型
浏览数
下载数
收藏数
```

## 12. 展示 Docker 部署结构

可以打开：

```text
docs/Docker部署教程.md
docker-compose.yml
```

说明：

```text
项目通过 Docker Compose 一次性启动 Nginx、前端、后端、MySQL、Redis、MinIO。
Nginx 负责前端静态资源和 /api 反向代理。
MySQL 保存业务数据，Redis 用于缓存，MinIO 用于资料文件存储。
```

## 13. 推荐讲解顺序

正式展示时可以这样讲：

```text
1. 介绍项目背景：课程资料共享和互助问答平台。
2. 展示 Docker 一键启动。
3. 展示前端首页。
4. 展示健康检查，证明依赖服务正常。
5. 登录 student。
6. 发布资料。
7. 展示 AI 审核通过。
8. 展示资料列表和详情。
9. 展示问答或后台管理。
10. 展示文档和部署教程。
```

## 14. 演示失败时的应急排查

如果页面打不开：

```powershell
docker compose ps
docker compose logs -f nginx
```

如果接口失败：

```powershell
docker compose logs -f backend
```

如果数据库异常：

```powershell
docker compose logs -f mysql
```

如果数据不对或乱码：

```powershell
docker compose down -v
docker compose up -d --build
```

注意：`down -v` 会删除本地 Docker 数据，只适合演示环境重置。
