# 任务分工与进度

本文档用于记录小组成员分工、当前进度和后续任务。负责人应在每次阶段推进后更新本文档。

## 1. 角色分工建议

| 角色 | 主要职责 |
| --- | --- |
| 项目负责人 | 管理 `main` / `dev` 分支、审核 PR、控制进度、维护部署文档 |
| 后端负责人 | 维护 Spring Boot 接口、业务逻辑、数据库访问 |
| 前端负责人 | 维护 Vue 页面、接口调用、交互体验 |
| 数据库负责人 | 维护表结构、初始化 SQL、测试数据 |
| 测试负责人 | 维护冒烟测试、接口验证、演示检查 |
| 文档负责人 | 维护 README、部署教程、演示流程、项目说明 |

实际人数较少时，一个人可以负责多个角色。

## 6. 提交前检查

负责人在合并到 `main` 前执行：

```powershell
git status
docker compose up -d --build
docker compose ps
powershell -ExecutionPolicy Bypass -File .\backend\scripts\api-smoke-test.ps1
```

确认：

```text
1. 工作区没有误提交文件
2. 所有核心容器 healthy
3. smoke test passed
4. README 和文档导航没有过期
```
